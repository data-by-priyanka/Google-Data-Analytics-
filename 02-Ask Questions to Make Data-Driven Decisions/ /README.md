# 02 – Ask, Prepare, Process

## Overview

This module focuses on the first three phases of the data analysis process:

1. Ask  
2. Prepare  
3. Process  

These stages form the foundation of any successful data project. Without a clear problem and clean data, analysis cannot produce reliable insights.

---

## 1. Ask Phase

### Purpose

The Ask phase defines the problem that needs to be solved.

Before analyzing data, it is important to understand:

- What is the business problem?
- Who are the stakeholders?
- What decision needs to be made?
- What success looks like?

### SMART Questions

Questions should be:

- Specific  
- Measurable  
- Achievable  
- Relevant  
- Time-bound  

Clear questions prevent confusion later in the analysis.

---

## 2. Prepare Phase

### Purpose

The Prepare phase involves collecting and organizing relevant data.

### Key Activities

- Identifying data sources
- Gathering datasets
- Understanding data structure
- Checking data credibility
- Ensuring data privacy and ethics

### Data Credibility Considerations

Reliable data should be:

- Accurate
- Complete
- Consistent
- Relevant
- Timely

Poor data quality leads to misleading insights.

---

## 3. Process Phase

### Purpose

The Process phase focuses on cleaning and transforming data so it becomes usable for analysis.

### Common Data Cleaning Tasks

- Removing duplicates
- Handling missing values
- Correcting formatting errors
- Standardizing text (e.g., INDIA vs India vs india)
- Validating data types

Clean data is essential for accurate analysis.

---

## Data Ethics and Privacy

Data analysts must:

- Respect user privacy
- Avoid bias in analysis
- Follow data protection regulations
- Use data responsibly

Ethical handling of data builds trust and credibility.

---

## Tools Used in These Phases

- Spreadsheets for cleaning and organizing
- SQL for querying datasets
- Basic data validation techniques
