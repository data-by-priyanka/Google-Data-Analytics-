## Spreadsheet Tasks

Spreadsheets help analysts perform two main types of work:

### 1. Organizing Data
- Sorting and filtering data
- Using pivot tables to summarize information

### 2. Calculating Data
- Creating formulas
- Using built-in functions

---

## Spreadsheets and the Data Life Cycle

Spreadsheets support every stage of the data life cycle:

**Plan**  
Set clear formatting rules, naming conventions, and organizational standards. This improves consistency and makes collaboration easier.

**Capture**  
Connect spreadsheets to data sources like surveys or databases so information updates automatically.

**Manage**  
Store, filter, organize, and secure data. Control who can access and edit it.

**Analyze**  
Use formulas, functions, and pivot tables to extract insights and summarize results.

**Archive**  
Store older versions of spreadsheets for historical reference.

**Destroy**  
Delete data when it is no longer needed, especially for legal or security reasons.

---

## Formulas

A formula is a custom calculation created by the user.

Formulas use operators such as:
- `+` for addition
- `-` for subtraction
- `*` for multiplication
- `/` for division

### Cell Reference

A cell reference identifies a specific cell using:
- Column letter
- Row number

Example: `A1`

A range includes multiple cells:
- `A1:A10`

Cell references automatically update when copied unless locked.

---

## Auto-Fill

The fill handle (small square at the bottom-right of a cell) allows you to:

- Copy formulas down a column
- Copy formulas across a row
- Create number sequences automatically

This saves time and reduces manual errors.

---

## Relative, Absolute, and Mixed References

**Relative Reference (A1)**  
Changes when copied.

**Absolute Reference ($A$1)**  
Does not change when copied.

**Mixed Reference ($A1 or A$1)**  
Locks either the row or the column.

Press `F4` to switch between reference types.

---

## Data Range

A data range is the group of cells used in a formula.

Example:
`=SUM(A1:A10)`

Highlighting a formula often shows colored boxes around the referenced cells.

---

## Functions

A function is a built-in command that performs a specific task.

Example:
- `SUM()` adds values
- `AVERAGE()` calculates mean
- `COUNT()` counts entries
- `COUNTIF()` counts based on a condition

### Formula vs Function

- A **formula** is custom-written.
- A **function** is predefined.

Functions save time and reduce manual calculation errors.

---

## Problem Domain

A problem domain is the specific area being analyzed.

Clearly defining the problem domain helps:
- Avoid confusion
- Reduce wasted time
- Set clear boundaries

---

## Structured Thinking

Structured thinking means breaking a problem into clear steps:

1. Identify the problem
2. Organize information
3. Identify gaps
4. Explore possible solutions

Structured thinking reduces mistakes and improves clarity.

---

## Scope of Work (SOW)

A Scope of Work outlines the expectations and boundaries of a project.

It usually includes:

### Deliverables
What will be completed and provided?

### Timeline
When will tasks and milestones be completed?

### Milestones
Major checkpoints that show progress.

### Reports
How and when updates will be shared.

An SOW ensures everyone agrees on:
- What is being done
- When it will be done
- How success will be measured

---

## Context

Context gives meaning to data.

To understand data properly, consider:

- **Who** created or collected it?
- **What** does it impact?
- **Where** was it collected?
- **When** was it collected?
- **Why** was it collected?
- **How** was it collected?

Without context, data can be misunderstood or misused.

To ensure fairness:
- Use representative samples
- Avoid bias
- Ask critical questions about the source
- Collect data objectively

---

## Key Takeaway

Strong spreadsheet skills support every stage of the data life cycle.

Clear problem definition, structured thinking, and proper context turn raw data into reliable insights.
